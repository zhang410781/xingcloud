# 安全策略

Xing-Cloud 涉及运维任务、凭据配置、外部系统接入和智能体工具调用，安全问题请优先私下反馈，不要直接公开可利用细节。

## 反馈安全问题

如果你发现漏洞或敏感信息泄露风险，请通过项目维护者公开提供的联系方式反馈。反馈时建议包含：

- 受影响的版本、分支或提交。
- 复现步骤和影响范围。
- 相关日志、截图或最小复现请求。
- 你认为可行的修复建议。

在维护者确认并给出处理方案前，请不要公开利用方式、真实环境地址、Token、密钥或客户数据。

## 生产部署基线

- 设置强随机 `SECRET_KEY`。
- 设置 `DEBUG=0`。
- 明确配置 `ALLOWED_HOSTS`，不要在公网使用通配值。
- 关闭不需要的跨域访问，生产环境建议设置 `CORS_ALLOW_ALL_ORIGINS=0`。
- 修改或禁用演示账号和默认密码。
- 使用独立的 MySQL、Redis 账号和强密码。
- 妥善保管云账号、Kubeconfig、SSH 密钥、Grafana Token、模型供应商 API Key。
- 对任务执行、WebSocket、菜单、按钮和 API 访问保持后端 RBAC enforcement。
- 定期清理运行日志中的敏感信息。

## 不应公开的内容

- 真实生产主机、数据库、集群、云资源和客户名称。
- 真实密钥、Token、Cookie、Session、Kubeconfig、SSH 私钥。
- 可直接用于入侵或横向移动的漏洞细节。
