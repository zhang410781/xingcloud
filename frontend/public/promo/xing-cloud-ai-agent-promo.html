SCENARIO

典型场景：生产 K8S 集群异常研判

❌ 人工排障路径

  1. 收到告警通知 → 登录告警中心看红点
  2. 切换到 Grafana 看节点指标趋势
  3. 切换到 ClickHouse 查异常 Pod 日志
  4. 切换到 K8S Dashboard 查 Pod 状态
  5. 切换 CMDB 查应用归属
  6. 找值班同学问最近有什么变更
  7. 手动总结 → 写群消息通知

⏱ 平均耗时：15+ 分钟 · 4+ 系统切换

✅ AI Agent 处理路径

  1. 用户提问：“巡检当前集群并分析异常 Pod”
  2. Agent 并行查询：告警 / 指标 / 日志 / CMDB
  3. 确定性采集：节点状态 · Pod 重启 · Warning 事件 · 异常日志
  4. 模型在证据置信度上限内归纳候选根因与影响范围
  5. 生成结论 + 建议操作 + 巡检任务草稿
  6. 用户确认 → 任务进入任务中心执行
  7. 全链路留痕可审计

⏱ 平均耗时：<3 分钟 · 1 次对话

从跨系统查 15 分钟，到一次对话 3 分钟 10 / 14

SAFETY

RBAC

权限管理

每位用户拥有独立角色和权限，Agent 调用工具受 RBAC 约束，越权操作自动拒绝。

TOOL WHITELIST

工具白名单

LLM 只能调用平台注册过的工具，真正执行由后端完成，模型不直接触碰基础设施。

ACTION CONFIRM

动作确认

写操作必须先由 Agent 生成草稿，用户确认后才进入任务中心执行，杜绝自动闯祸。

AUDIT

审计回放

每一次对话、工具调用、待确认动作、执行结果全链路留痕，可回放可追溯。
