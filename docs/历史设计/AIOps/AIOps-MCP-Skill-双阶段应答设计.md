# AIOps MCP + Skill 双阶段应答设计

> **历史设计，不代表当前产品能力。** 当前实现见 [AIOps 当前实现说明](../../AIOps智能体实现说明.md)。

## 背景

AIOps 智能助手需要同时满足两类目标：

- 由 LLM 理解自然语言并选择平台能力，避免关键词规则路由。
- 最终回复稳定、可读、可追溯，不因为 LLM 最后一轮自由发挥而丢失工具证据。

因此当前设计采用“双阶段 LLM + MCP + Skill + 代码兜底”的方式。

## 总体流程

1. 用户提出问题，例如“分析生产 workorder-center 最近异常”。
2. 第一阶段 LLM 只负责规划工具调用，通过 tool-calling 选择平台 MCP 工具。
3. 平台内置 MCP 或外部 MCP 返回结构化事实数据。
4. 后端生成一份代码兜底草稿，确保即使 LLM 二次整形失败也有可用答案。
5. 第二阶段 LLM 作为“回答整形器”，读取工具事实、兜底草稿与启用的 Skill 模板，生成最终中文回复。
6. 如果第二阶段 LLM 输出为空、太泛、缺少必要结构或调用异常，自动回退到代码兜底草稿。

## 平台内置 MCP 工具

平台功能以“平台内置 MCP”的方式暴露给 LLM，而不是通过用户问题关键词匹配路由。

当前平台内置 MCP 能力包括：

- 资源上下文 MCP：`query_task_resources`
- 可观测性 MCP：`query_observability`
- 工单系统 MCP：`query_workworkorders`
- 任务中心 MCP：`query_task_center`、`generate_host_task`
- 时间中心 MCP：`query_event_wall`
- 容器管理 MCP：`query_container_assets`
- 容器 MCP：`query_container_status`

LLM 看到的是工具 schema，并通过 tool-calling 决定调用哪个工具。后端只负责按 MCP 工具注册表执行被选中的工具，不根据用户原始问题做关键词路由。

## Skill 的职责

Skill 不负责直接查数据，而是约束 LLM 的行为和回答模板：

- `证据优先应答`：要求先调用 MCP 工具获取事实。
- `故障关联分析`：要求异常类问题组合告警、日志、链路和变更证据。
- `自动化安全护栏`：要求执行类任务先生成草稿并等待确认。
- `结构化应答模板`：要求二阶段整形基于工具事实输出结构化答案。

异常/故障类问题默认使用三段式：

```text
结论：
- ...

依据：
- ...

建议操作：
- ...
```

## 为什么需要二阶段 LLM 整形

如果只依赖第一阶段 LLM 的最终回复，可能出现以下问题：

- 模型已经调用了工具，但最终只回答“已定位到异常，请查看日志中心”。
- 工具事实很多，但模型没有提炼数量、影响点和建议动作。
- 模型把结构化查询串写成 `environment:prod service:xxx 日志 链路 变更`，导致工具参数噪音过多。

二阶段整形的输入是工具事实 JSON 和 Skill 模板，目标是“基于事实重新组织答案”，而不是自由发挥。

## 事实约束与兜底

二阶段整形必须遵守：

- 只能使用工具返回的数据。
- 不能编造告警数量、日志内容、Trace、发布版本或成本金额。
- 数据不足时必须明确说明“当前证据不足以确认”。
- 异常类问题必须包含“结论 / 依据 / 建议操作”。

如果二阶段 LLM 输出不合格，后端回退到代码兜底草稿。这样可以兼顾自然表达与稳定性。

## 结构化 Query 清洗

LLM 有时会把自然语言改写为结构化查询串，例如：

```text
environment:prod service:workorder-center 最近异常 告警 日志 链路 变更
```

这类串不能直接作为所有工具的全文检索条件，否则 `日志/链路/变更` 会在不同工具中互相干扰。

当前做法是：

- 保留可用过滤信息，例如 `environment=prod`、`service=workorder-center`。
- 清洗跨工具噪音词，例如日志工具忽略“链路/变更”，链路工具忽略“日志/变更”。
- 再把清洗后的参数传给 MCP 工具执行。

## 设计边界

这套设计不是让代码替代 LLM 回答，而是分层：

- LLM 第一阶段：理解问题并选择工具。
- MCP 工具：返回平台事实。
- LLM 第二阶段：按 Skill 模板整形答案。
- 代码兜底：在二阶段失败时保证可用、可信的回复。

不同类型问题的模板可以持续配置化，例如：

- 工单汇总模板
- 告警处置模板
- 工单汇总模板
- K8s 异常模板
