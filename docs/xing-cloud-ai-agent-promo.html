Agent‑4 风格的告警精准研判

告警先通知，持续两分钟后再基于当时已经产生的真实证据执行一次精准研判。

确定性采集

  • 告警前十分钟至执行时刻的指标趋势与峰值
  • 目标 Pod describe 等价状态和 Warning Event
  • 当前日志与 previous 崩溃日志
  • 重启增量、OOM、Request/Limit、节点磁盘和 I/O
  • 同一窗口内的发布、任务与变更证据

受约束的结论

  • 确定性故障模式先生成候选根因
  • 模型只整理已有证据，不提出无证据原因
  • 单来源最高 49%，两类来源最高 79%
  • 证据不足明确输出“无法确诊”
  • 修复只生成待确认任务草稿
