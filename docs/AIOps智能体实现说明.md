# AIOps 当前实现说明

> 状态：当前技术实现
> 历史演进方案见 [历史设计/AIOps](历史设计/AIOps/)。

## 1. 定位

Xing-Cloud AIOps 是现有 Django 平台中的智能运维能力层。它把业务上下文、资产、告警、指标、日志、K8S、任务和工单转换为受权限控制的查询与动作，并使用模型整理证据和生成可读答复。

当前不存在独立部署的巡检 Agent、告警 Agent、日志 Agent 或变更 Agent。Agent‑1/2/3/4 仅用于描述告警研判内部逻辑职责。

## 2. 核心模型

- `AIOpsKnowledgeEnvironment`：业务上下文。
- `AIOpsChatSession`、`AIOpsChatMessage`：会话和消息。
- `AIOpsActionDefinition`：Action Registry。
- `AIOpsSkill`：内置和自定义 Skill。
- `AIOpsMCPServer`：外部 MCP 配置。
- `AIOpsModelProvider`：模型供应商。
- `AIOpsRunbook`、`AIOpsRunbookVersion`：Runbook 与版本。
- 智能体审计及待确认动作相关模型。

## 3. 业务上下文

上下文保存一级业务分组、Prometheus 和日志源单值绑定。K8S 集群由业务分组下登记的 K8S 资产推导。指标源和日志源允许被多个上下文复用。

智能助手、巡检和告警研判只接收上下文 ID，再由服务端解析资源，不使用默认数据源或全局唯一集群进行静默兜底。

## 4. Action、Skill 与 MCP

### Action

Action 决定问题类别、是否需要上下文、风险级别、处理模式、默认 Skill 和输出块。它负责路由，不直接代表一个模型实例。

### Skill

Skill 保存专业步骤、适用 Action、工具依赖、风险和输出契约。内置 Skill 以服务代码中的定义初始化，自定义 Skill 保存到数据库。

### MCP

平台 MCP 将已有查询能力以工具形式暴露；外部 MCP 通过配置接入。最终工具集合同时受 MCP 可用性、Skill 声明、Action 风险和当前用户 RBAC 过滤。

## 5. 确定性处理路径

以下问题优先由后端确定性流程处理，而不是等待模型规划工具：

- 当前集群节点数。
- 集群综合巡检和服务器巡检。
- 节点、工作负载、Pod 或服务诊断。
- 首次告警通知后的即时精准研判。

确定性流程将结果保存为结构化报告块和 Markdown，模型不可用时仍能返回部分结果。

## 6. 告警精准研判

规则启用自动研判后，首次通知完成即进入逻辑流水线：

1. 采集 `starts_at - 10 分钟` 到执行时刻的指标。
2. 读取目标 Pod describe 等价状态和 Warning Event。
3. 读取当前日志与 previous 日志。
4. 查询告警开始到执行时刻的日志和变更。
5. 生成确定性候选根因和置信度上限。
6. 使用模型整理已有证据。
7. 保存 `AlertAnalysis` 并发送独立研判通知。

研判任务由同一应用镜像中的 `run_alert_analysis_worker` 独立轮询数据库队列。它与统一 Scheduler 分离运行，以保证规则求值较慢时仍能及时开始研判，但不增加独立 Agent 服务或新的数据契约。

单一来源最高 49%，两类来源最高 79%，三类来源加直接证据才允许超过 80%。

## 7. 模型调用

模型供应商支持 Base URL、API Key、默认模型、备用模型、超时、最大 token 和模型列表发现。管理员保存并测试供应商后，还需在智能体策略中选择默认供应商。

模型的职责是：

- 理解普通自然语言问题。
- 在允许的场景选择工具。
- 整理平台已经提供的证据。
- 生成结构化答复或待确认动作。

模型不能补造指标、日志、Trace、K8S 状态或执行结果。

## 8. 停止生成与超时

异步消息状态为 pending、running、completed、failed 或 cancelled。取消接口只允许会话所有者或管理员操作，并具有幂等性。

总处理时限为五分钟。各阶段在外部调用前后检查取消状态；已经取消的消息不会被晚到结果覆盖。

## 9. Runbook

Runbook 位于智能体配置页，支持草案、从会话生成、版本、发布、归档、删除和复盘知识沉淀。它是知识与流程资产，不是独立 Agent，也不默认自动执行。

## 10. 安全边界

- 查询和写操作都受当前用户权限控制。
- 写操作先生成待确认动作。
- 敏感字段在证据和通知中脱敏。
- 会话、工具、模型和动作进入智能体审计。
- 不展示模型思维链。

## 11. 当前未提供

- APM、Trace 和服务调用链。
- 独立多 Agent 微服务。
- 无人值守自动修复。
- 依赖模型才能执行的巡检。
