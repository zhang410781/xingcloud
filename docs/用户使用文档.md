# Xing-Cloud 用户使用文档

> 适用产品：Xing-Cloud 运维智能体平台
> 适用场景：生产运维

## 1. 推荐初始化顺序

新环境不预置资产和监控数据，请按以下顺序配置：

1. 在 `资产管理 / 资产登记` 创建一级业务分组并登记服务器。
2. 在 `平台管理 / K8S 集群` 接入已有集群。
3. 将该集群登记为一级业务分组下的 K8S 资产。
4. 在 `资产管理 / 中间件资产` 登记真实中间件。
5. 在 `可观测性 / 数据源` 添加 Prometheus 和日志源并测试连接。
6. 在 `AIOps / 知识图谱` 的配置页创建业务上下文。
7. 检查上下文就绪状态后，再配置告警规则、通知和巡检计划。

## 2. 业务上下文

业务上下文统一监控、日志、告警、K8S、资产、巡检和智能助手的数据范围。

创建时需要：

- 显示名称、唯一编码和业务线。
- 一个 CMDB 一级业务分组。
- 一个 Prometheus。
- 一个日志源。

K8S 集群由所选资产分组中的 K8S 资产自动带入。指标源和日志源允许被多个业务上下文复用。唯一编码创建后不能修改，外部告警的 `environment` 必须与其完全一致。

顶部业务上下文选择器用于多个模块的默认范围。页面中的局部筛选只在当前上下文内生效。

## 3. 当前菜单

### 运行概览

查看平台整体资产、告警、任务、工单和风险态势。它不是 K8S 指标大屏；详细指标请进入可观测性监控看板。

### AIOps

- **智能助手**：查询、巡检、日志分析、告警研判和任务草稿。
- **知识图谱**：查看业务上下文关系并管理上下文绑定。
- **智能体配置**：管理策略、Action、Skill、MCP、模型供应商和 Runbook。
- **智能体审计**：查看会话、工具、模型和动作记录。

生成时间过长时点击“停止生成”。单次请求最长处理五分钟，超时会保留部分结果。

### 可观测性

- **平台总览**：数据源、集成和整体监控摘要。
- **告警中心**：查看、认领、静默、升级、研判和恢复告警。
- **告警规则**：管理模板和规则实例。
- **监控看板**：K8S、服务器、数据库、中间件、日志五类固定看板。
- **指标查询**：选择 Prometheus 执行指标查询。
- **日志查询**：查询 Loki、Elasticsearch 或 ClickHouse。
- **数据源**：管理指标源和日志源。

固定日志看板仅使用 Elasticsearch 或 ClickHouse；Loki 在日志查询页使用。

### 任务中心

任务工作台用于创建、确认、执行、取消和重跑受控任务。计划任务菜单是否显示取决于部署配置。

### 资产管理

- **资产登记**：一级业务分组、服务器和 K8S 资产。
- **中间件资产**：数据库和中间件连接信息。

“未检测”表示尚未执行检测；只有最近检测失败才表示连接异常。

### 工单系统

- **应用发布**：K8S 发布、审批、分批、日志、停止、重跑和回滚。
- **事务工单**：通用运维事项的申请与处理。
- **审批流**：配置应用发布审批步骤。

### 平台管理

当前只提供 K8S 集群管理。Docker/容器环境不是当前菜单功能。

### 系统管理

管理用户、角色、用户组、权限、模块可见性和操作审计。

## 4. 告警规则和通知

### 规则

推荐从内置模板创建规则实例并选择当前上下文使用的 Prometheus。保存前先试运行：

- `matched_count=0` 表示当前查询没有返回匹配序列。
- `would_fire_count=0` 表示没有对象满足触发条件。
- 无数据次数持续增加时，检查数据源、指标名、标签和 exporter。

### 通知资源

通知资源包括渠道、接收人、接收组和巡检报告计划。接收人只需填写名称和对应渠道联系方式，不需要平台用户 ID。

通知策略负责标签匹配、分组、静默、抑制、升级、恢复通知和研判通知。飞书机器人需要签名密钥。

## 5. 巡检报告

在告警规则的通知资源中新增巡检报告计划：

1. 选择业务上下文。
2. 选择集群综合巡检或服务器巡检。
3. 选择每日/每周、时间和时区。
4. 选择通知渠道和接收资源。
5. 保存，或点击立即发送。

立即发送会实时采集指标、K8S、日志和事件。结果在平台保存结构化报告，并向渠道发送 Markdown。

## 6. 日志源配置

日志字段映射应指向数据中真实存在的字段，常用规范字段包括：

- `timestamp`
- `message`
- `level`
- `service`
- `namespace`
- `pod`
- `container`
- `host`

ClickHouse 可以配置多个集合，例如容器日志、K8S Event 和 Ingress 访问日志。Elasticsearch 可以配置索引模式，不要求只能使用 `k8s-*`；应按实际采集索引填写。

## 7. 权限建议

对外开放只读用户时，只授予查看和查询权限，不授予 manage、execute、approve、终端或配置权限。完成角色配置后同时验证页面按钮和直接 API 写请求均被拒绝。

## 8. 当前产品边界

- 不提供 APM、Trace 和服务调用链。
- 不提供拖拽式看板编辑器。
- 不自动修复告警或自动回滚。
- 不生成 PDF 巡检报告。
