# K8S 平台管理 — 产品工作流

## 1. 产品目标

K8S 集群管理让 SRE 数字员工能够感知和操作容器平台。故障研判智能体和日志分析智能体需要获取 K8S Pod/Node 事件才能做根因分析；巡检智能体需要 K8S 集群健康状态才能产出报告。

## 2. 工作流

### 2.1 接入 K8S 集群

```
平台管理 → 新增集群
   → 填写集群名称、API Server 地址
   → 系统生成 kubeconfig 引导命令（只读 ServiceAccount）
   → 用户在集群中执行引导命令，获得 kubeconfig
   → 粘贴 kubeconfig 内容 → 保存
   → 系统验证连通性
   → 接入成功
```

### 2.2 日常管理

```
查看集群状态：Node 列表、Pod 列表、命名空间、事件
容器管理：查看容器日志、进入容器、重启容器
镜像管理：查看镜像列表、删除镜像
```

### 2.3 固定巡检与排障

K8S固定巡检通过 `k8s.inspect` Action 进入确定性证据管线，不依赖模型主动调用工具。用户只选择业务上下文，平台自动使用其绑定的唯一K8S集群、Prometheus和日志源。

| Profile | 使用场景 | 核心证据 |
|---------|----------|----------|
| cluster | 巡检整个集群 | Node、Pod、工作负载、PVC、资源余量、API Server |
| node | 检查指定节点 | Ready、压力状态、CPU、内存、负载、磁盘、inode、网络、I/O、kubelet |
| workload | 检查Pod或工作负载 | 副本、调度、退出原因、探针、Limits、Warning Event、当前及上次崩溃日志 |
| service | 检查服务 | Service、Endpoint、Ingress、后端Pod、请求量、错误率、延迟、最近五分钟日志 |
| control_plane | 检查控制平面 | API Server、ETCD、Scheduler、Controller Manager、CoreDNS、审计日志 |

固定巡检输出健康分、异常项、关键证据、建议操作、未获取证据和监控/日志/告警/资产下钻入口。健康分按严重问题20分、警告8分、提示2分透明扣分；同一对象的同一问题类型只扣一次，日志样本数量不直接叠加扣分。写操作只能生成待确认任务草稿。

### 2.4 K8S 告警规则模板

内置规则模板（Codex 当前正在开发的）：

| 模板 | 严重级别 | 条件 |
|------|---------|------|
| Pod CrashLoopBackOff | critical | 重启次数 > 3/5min |
| Node NotReady | critical | 节点状态 != Ready 持续 > 5min |
| Pod Pending 超时 | warning | Pending 状态 > 10min |
| 集群 OOM | critical | OOMKilled 事件 > 3 次/5min |
| Ingress 5xx 突增 | warning | 5xx 比例 > 5% |

## 3. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| K8S API | 平台管理页面 |
| K8S 指标+事件 | 故障研判智能体（Pod 事件证据） |
| K8S 集群 | 巡检智能体（集群健康状态） |
| K8S 日志源 | 日志分析智能体（容器日志） |
