# 任务中心与 Runbook

> 状态：任务中心和 Runbook 管理当前已实现；计划任务入口受配置开关控制

## 1. 任务中心

任务中心承接受控的主机与 K8S 操作，包括命令、脚本、Ansible、K8S 任务和由智能助手生成的待确认任务。

任务执行必须经过权限检查，保存目标、参数、创建人、执行状态、输出和错误。高风险动作不能仅凭模型回答直接执行。

## 2. 任务工作流

1. 用户手动创建任务，或智能助手生成任务草稿。
2. 平台校验目标、权限、参数和风险。
3. 用户确认后执行。
4. 平台记录逐目标结果、错误和审计信息。
5. 可对符合条件的任务重跑或取消。

## 3. 计划任务

后端支持周期计划、下一次执行时间预览、启停、立即运行和执行历史。前端“计划任务”是否显示由部署配置开关控制，因此不能假定所有环境都开放该菜单。

## 4. Runbook 手册

Runbook 位于 `AIOps / 智能体配置` 的编排区域，不是独立一级菜单或独立智能体。

当前支持：

- 手工生成草案。
- 从事故会话生成草案。
- 编辑内容、环境、服务、标签和引用来源。
- 发布、归档、删除和查看版本历史。
- 发布后沉淀为复盘知识。

Runbook 内容可以作为处理建议或任务草稿来源，但当前没有通用的“发布后自动一键执行全部步骤”产品闭环。

## 5. 当前边界

- 巡检报告使用独立确定性证据管线，不依赖任务中心执行巡检脚本。
- Runbook 不自动修复告警。
- 执行能力受目标主机/K8S 连接、权限和部署依赖限制。
