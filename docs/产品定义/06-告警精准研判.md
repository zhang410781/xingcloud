# 告警精准研判

> 状态：当前已实现
> 架构：单服务内的逻辑阶段，不是独立 Agent 服务

## 1. 触发规则

规则可以通过 `auto_analyze` 控制是否自动精准研判：

1. 告警首次触发后立即发送原始告警摘要。
2. 首次通知成功或发送失败已经记录后，立即创建唯一研判任务。
3. 快速恢复不取消已经开始或排队的研判，报告标记“研判期间已恢复”。
4. 研判完成后只有在本活动周期的首次告警实际送达时，才向相同策略中实际送达的渠道发送独立更新；恢复通知继续走原流程。

同一告警指纹只保留一个等待或运行中的研判任务。手动研判不受自动范围限制。

研判任务与研判通知相互独立：静默、屏蔽、抑制、分组等待、分组聚合或风暴降噪不会阻止证据采集和详情入库，但会阻止研判绕过原告警单独通知。告警恢复后再次触发时按新的活动周期重新判断，历史周期的首次通知不能作为本周期研判通知的发送依据。

## 2. 逻辑阶段

状态依次为：

`queued_after_first_notification → collecting_metrics → collecting_k8s → collecting_logs → collecting_changes → synthesizing → completed/partial`

这些阶段对应 Agent‑1/2/3/4 的职责拆分，但全部运行在现有 Django 后端。规则求值由统一 Scheduler 执行，研判队列由同一镜像中的独立 Worker 每五秒消费，避免一轮 Prometheus 规则求值阻塞即时研判；它不是独立 Agent 服务。

Worker 会修复升级前遗留的异常状态：若活跃告警所属规则启用了自动研判，但告警只有 `pending` 标记且没有真实 `AlertAnalysis` 记录，则自动补建一次任务，后续按告警记录去重。

## 3. 指标证据

指标窗口为 `starts_at - 10 分钟` 到研判执行时刻。查询固定使用业务上下文绑定的 Prometheus，并按 namespace、Pod、container 和 node 精确过滤。指标证据包含状态、阈值、单位、时间窗、样本数和证据编号。

Pod/容器研判重点采集：

- 告警规则原始指标。
- 窗口内重启增量。
- CPU 与内存峰值。
- CPU Request、内存 Limit。
- OOMKilled 指标。
- 节点磁盘 I/O 和内存可用率。

节点类告警补充 CPU、内存、负载、磁盘、inode、I/O、网络错误及节点 Conditions。

生命周期累计重启只作为背景，不能代替时间窗口内的重启增量。

## 4. K8S 深度证据

平台通过只读 Kubernetes API 读取目标 Pod 的 describe 等价信息：

- phase、Ready、waiting/terminated reason、exit code 和重启次数。
- 镜像、Request、Limit 和所在节点。
- Warning Event。
- 当前日志和 previous 崩溃日志。

单项读取失败时记录原因并继续其他阶段，不依赖本地 `kubectl`。

## 5. 日志与变更

日志主窗口从告警 `starts_at` 开始，最多采集五分钟，并按 namespace、Pod、容器、服务等维度精确过滤。关键证据只保留 Error、Warning、异常堆栈、OOM、GC、连接、存储和启动失败等模式；普通成功访问日志不作为关键根因证据。

平台同时查询时间窗口内的发布、任务和变更记录，作为候选相关证据。

## 6. 根因与置信度

确定性模式优先识别 OOMKilled、探针失败、CrashLoop、镜像/配置错误、调度失败、PVC、节点和网络异常。每个候选结论必须引用平台生成的证据编号，模型不得创建新证据。

- 单一独立来源最高 49%。
- 两类独立来源最高 79%。
- 80% 以上需要至少三类独立来源和直接异常证据。
- 模型只能整理已有证据，并受平台计算的置信度上限约束。

无法形成证据链时固定说明“告警仍活跃但无法确诊”或“已恢复但无法确诊”，不得凭经验补充无证据原因。

## 7. 输出

网页 `/observability/alerts/:id` 在登录和 `ops.alert.view` 权限校验后展示告警概览、时间线、指标、K8S 状态、日志与事件、候选根因、反向证据、置信度和 P0/P1/P2 建议。飞书等渠道发送摘要和详情链接，不展示模型思维链、完整 PromQL 或无限量原始日志。

单一已知 Warning 采用定向研判；Critical、未知资源类型、核心控制面服务或五分钟内同范围不少于三条关联告警采用完整研判。

## 8. 当前边界

- 不保存独立 K8S 现场快照。
- 不等待告警未来二十分钟的数据。
- 不自动执行修复，只能生成需人工确认的任务草稿。
