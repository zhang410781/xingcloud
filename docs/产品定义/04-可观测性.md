# 可观测性

> 状态：当前已实现；部分后端看板接口属于内部能力

## 1. 当前入口

可观测性菜单包含平台总览、告警中心、告警规则、监控看板、指标查询、日志查询和数据源。

## 2. 数据源

### 指标

当前指标查询、固定看板、告警和巡检以 Prometheus 为核心。数据源支持连接检测、启停和健康状态记录。

### 日志

数据源目录支持：

- Loki：用于日志查询。
- Elasticsearch：用于日志查询、业务上下文证据和固定日志看板。
- ClickHouse：用于日志查询、业务上下文证据、固定日志看板和已配置的日志集合。

日志源必须配置规范化字段映射，至少保证时间、消息、级别、服务、命名空间和 Pod 等实际存在的字段能够映射；字段缺失时对应查询或证据显示可诊断空态。

## 3. 固定原生看板

| 入口 | 当前内容 |
| --- | --- |
| K8S 集群 | 节点、Pod、命名空间、CPU/内存、磁盘、网络、I/O、API Server、Requests/Limits、PVC |
| 服务器 | CPU、内存、负载、磁盘、网络和节点排行/趋势 |
| 数据库 | MySQL、PostgreSQL 专属指标 |
| 中间件 | Redis、Kafka 专属指标 |
| 日志 | 总量、错误量、错误率、服务数、趋势、分布、排行和异常日志 |

看板使用 24 列布局和浅色界面，数据随当前业务上下文及所选节点、命名空间、时间范围更新。

固定日志看板只接受 Elasticsearch 或 ClickHouse；Loki 继续用于日志查询页。

## 4. 用户可见与内部接口

### 用户可见

- 固定五类看板及筛选器。
- 指标查询和日志查询。
- 数据源增删改、连接检测和健康状态。

### 内部能力

后端保留看板定义 CRUD、导入、导出和查询接口，用于维护内置看板。当前前端不提供看板编辑器、拖拽布局或 JSON 导入入口。

## 5. 当前边界

- 不嵌入 Grafana，也不依赖 Grafana 才能展示固定看板。
- 不提供 APM、Trace 和服务调用链。
- exporter 或字段缺失只影响相关面板，不应导致整页失败。
