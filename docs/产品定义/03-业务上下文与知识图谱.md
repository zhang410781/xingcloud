# 业务上下文与知识图谱

> 状态：当前已实现

## 1. 产品定义

业务上下文是平台统一的数据范围。用户选择上下文后，监控、日志、告警、巡检、资产和智能助手使用同一组业务资源，避免各页面分别选择不一致的环境。

## 2. 创建条件

创建业务上下文必须选择：

- 一个已存在的 CMDB 一级业务分组。
- 一个已启用的 Prometheus 指标数据源。
- 一个已启用的日志数据源。

K8S 集群不在向导中重复选择，而是从一级业务分组下登记的第一项有效 K8S 资产自动推导。没有 K8S 资产时，上下文仍可保存，但 K8S 检查会显示未就绪。

## 3. 字段规则

| 字段 | 规则 |
| --- | --- |
| 显示名称 | 用户可读名称，必须唯一 |
| 唯一编码 | 小写字母、数字和连字符；创建后不可修改 |
| 业务线 | 用于说明业务归属 |
| 指标源 | 单值绑定，允许被多个上下文复用 |
| 日志源 | 单值绑定，允许被多个上下文复用 |
| 资产分组 | 单值绑定，范围内可包含多项资产 |
| 告警环境 | 必须与上下文编码完全一致 |

当前界面固定按生产类型保存，不向用户提供生产/测试/开发分层选择。

## 4. 配置向导

1. 基本信息：填写名称、编码、业务线、负责人和描述。
2. CMDB 资产：选择一级业务分组并预览资产、K8S 和中间件。
3. 数据源与告警：选择 Prometheus 和日志源，确认告警环境编码。
4. 确认：保存后执行绑定检查，并切换为当前上下文。

“发现绑定”只会填入唯一匹配的候选；存在多个候选时必须人工确认。

## 5. 健康检查

检查项包括指标源、日志源、K8S 资产、资产分组以及告警环境一致性。页面显示“已就绪”或“待完善”，失败项提供具体原因。

## 6. 知识图谱

图谱由平台根据业务上下文、资产、K8S、数据源、告警、发布和任务关系构建，用于：

- 展示业务资源关联。
- 限定智能助手和研判的数据范围。
- 补充资产与拓扑证据。

知识图谱不是 CMDB 录入入口，也不负责自动创建数据源或资产。

## 7. 当前边界

- 上下文内只选择一个 Prometheus 和一个日志源，但同一个数据源可以供多个上下文复用。
- 未匹配上下文的告警不静默回退到默认资源。
- 图谱关系质量取决于 CMDB 登记和资源标签，不推断不存在的调用链。
