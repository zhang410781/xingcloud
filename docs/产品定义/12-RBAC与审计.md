# RBAC 与审计

> 状态：当前已实现

## 1. RBAC

系统通过用户、角色、用户组、权限和模块可见性控制页面与接口访问。前端隐藏无权入口，后端仍会独立校验权限，不能依赖前端控制作为安全边界。

权限覆盖资产、任务、K8S、告警、数据源、日志、发布、工单、AIOps 配置和审计等模块。

## 2. 只读用户

开放只读账号时，应只授予对应模块的 `view`、查询或只读权限，不授予 `manage`、`execute`、`approve`、终端和配置权限。

只读角色验收至少检查：

- 能查看授权页面和数据。
- 不能新增、编辑、删除、审批或执行任务。
- 不能进入 WebShell、Pod Terminal 或修改告警/模型配置。
- 直接调用写接口同样返回无权限。

## 3. 管理员账号

管理员密码可以通过正常用户管理或密码修改流程更新。系统不应把 root/admin 密码硬编码为不可修改状态。

## 4. 审计

### 操作审计

记录用户对资产、任务、告警、发布、工单、K8S 和系统配置的关键操作。

### 智能体审计

记录会话、Action 路由、Skill、MCP 工具调用、模型调用、待确认动作、取消和结果状态。

审计记录不得展示完整 API Key、密码、Token 或通知签名密钥。

## 5. 当前边界

- 模块隐藏只影响菜单，不替代权限配置。
- MCP 和智能助手不能绕过当前用户权限。
- 审计用于追溯，不等同于外部 SIEM 或合规归档系统。
