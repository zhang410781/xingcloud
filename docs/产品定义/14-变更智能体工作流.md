# 变更智能体 — 产品工作流

## 1. 产品目标

变更智能体为每一次应用发布或基础设施变更提供全生命周期的安全保障。它解决 SRE 最担心的问题——"这次变更会不会出问题？"

在变更前，它评估风险：影响范围、历史关联故障、当前系统健康度。在变更中，它盯着关键指标：错误率、延迟、资源使用是否偏离基线。在变更后，它给出对比报告：有没有引入回退信号。

## 2. 工作流

### 2.1 变更前风险评估

```
工单审批通过后自动触发（或手动发起）
   → 获取变更范围：变更什么服务/配置/集群
   → 查询 CMDB：受影响的所有关联资产（上下游依赖、共享资源）
   → 查询历史告警：该服务/关联资产过去 7 天的告警记录
   → 查询当前健康度：该服务当前指标是否有未恢复的问题
   → 生成风险评估报告：
      - 影响范围列表（直接 + 间接）
      - 历史风险评分（该服务是否经常出故障）
      - 当前健康状态（绿灯 / 黄灯 / 红灯）
      - 建议操作：进行中 / 需审批补充 / 建议延期
   → 推送报告给变更执行人和审批人
```

### 2.2 变更中实时监控

```
变更开始执行
   → 自动锁定该服务的监控看板为"变更模式"
   → 每 30s 轮询关键指标：
      - 服务级别：错误率、P95 延迟、请求量
      - 基础设施：CPU/内存/磁盘
      - K8S：Pod 重启次数、Pending Pod
   → 如果检测到异常偏离（基于基线 ±3σ 或错误率 > 前 1h 均值 2x）：
      → 标记为"变更中异常"
      → 发送提醒到变更执行群
      → 可选：触发故障研判智能体简要分析
```

### 2.3 变更后基线对比

```
变更完成（或回滚）
   → 对比变更前后的指标基线（各取 15min 窗口）
   → 逐项生成对比报告：
      - 错误率：前 0.1% → 后 0.3% 🔴 上升
      - P95 延迟：前 120ms → 后 125ms 🟢 正常
      - 请求量：前 2000rps → 后 2100rps 🟢 正常
   → 总体结论：通过 / 需关注 / 建议回滚
   → 报告推送到工单评论区 + 变更执行群
```

## 3. 风险评估报告结构示例

```json
{
  "change_id": 1024,
  "service": "payment-order",
  "risk_level": "medium",
  "impact_scope": {
    "direct": ["payment-order"],
    "indirect": ["payment-gateway", "order-api"]
  },
  "health_check": {
    "status": "yellow",
    "current_alerts": [
      {"name": "payment-order 5xx > 1%", "severity": "warning", "since": "30 min ago"}
    ]
  },
  "history": {
    "incidents_last_7d": 2,
    "change_failure_rate": "5%",
    "last_change": "2026-07-15 - 回滚（OOMKilled）"
  },
  "recommendation": "建议先解决当前 warning 告警再执行变更"
}
```

## 4. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 工单系统（变更申请） | 变更智能体触发入口 |
| CMDB（资产关联关系） | 影响范围推导 |
| 可观测性（历史+实时指标） | 风险评估 + 实时监控 + 基线对比 |
| 告警引擎 | 变更中异常信号检测 |
| 变更智能体输出 | 通知（推送报告）+ 工单（报告追加） |

## 5. 常见场景

| 场景 | 说明 |
|------|------|
| 核心服务 payment-order 发版 → 变更前自动评估风险 | 发现该服务过去 7 天有 2 次故障，且当前有一个 warning 告警未恢复 → 建议先处理 |
| 发布中错误率突增 3x → 实时提醒 | 变更智能体发现异常信号 → 推送群消息 → SRE 快速决策回滚 |
| 发布后 15min 对比基线 | 发现 P99 延迟从 200ms 升到 350ms → 标记"需关注" → 开发排查代码问题 |

## 6. 边界与限制

- 当前变更智能体不自动执行回滚，仅检测并建议
- 影响范围推导依赖 CMDB 中定义的 CI 关系，缺失则范围不完整
- 历史风险评分基于告警数量，不包含人为故障定级
- 变更中监控仅支持 Prometheus 指标，不支持日志异常实时监测
