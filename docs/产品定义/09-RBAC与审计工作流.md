# RBAC 与审计 — 产品工作流

## 1. 产品目标

RBAC + 审计是 SRE 数字员工的"安全围栏"。它确保每个操作都来自正确的人、拥有正确的权限、留下完整的记录。7 个智能体的所有调用、所有 MCP 工具执行、所有 Action 确认/取消都需经过审计。

## 2. 权限模型

```
用户（User）
  └── 角色（Role）
       └── 权限码（Permission Code）
            ├── 后端 API 权限（views.py 中 @permission_required）
            ├── 前端路由/菜单/按钮权限（router 中 meta.permission）
            └── WebSocket 权限（Channels consumer 检查）
```

## 3. 审计范围

所有以下操作都记录到审计日志：

| 类型 | 记录内容 |
|------|---------|
| API 请求 | 用户、时间、请求路径、方法、参数、响应码 |
| AIOps 对话 | 用户提问、LLM 调用记录、MCP 工具调用参数与结果、最终回复 |
| Action 操作 | Action 类型、参数、创建/确认/取消时间、操作人 |
| Skill 调用 | 启用的 Skill、执行结果 |
| 任务执行 | 任务 ID、目标主机、命令、执行结果 |
| 登录/登出 | 用户、时间、IP、成功/失败 |

## 4. 工作流

### 4.1 用户与角色管理

```
平台管理员
   → 创建用户（用户名/密码/邮箱）
   → 创建角色（名称、权限码列表）
   → 为用户分配角色
   → 用户登录后自动加载权限
```

### 4.2 模块可见性控制

```
模块管理
   → 按角色控制左侧菜单的可见性
   → 未授权的路由自动跳转到 403 页
```

### 4.3 审计追溯

```
操作审计 / AIOps 审计
   → 按用户/时间/模块筛选
   → 查看操作详情
   → 可追溯：这个告警谁确认了？这个 Action 谁取消了？
```

## 5. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 用户管理 | 登录鉴权、权限校验 |
| 所有 API 调用 | 操作审计日志 |
| AIOps 引擎 | AIOps 审计日志 |
| 任务中心 | 任务执行审计日志 |

## 6. 边界与限制

- 不支持 SSO/LDAP/OAuth 登录（当前仅本地密码认证）
- 不支持细粒度资源级权限（如：只能查看某个分组的资产）
- 不支持审计日志导出
- 不支持审计日志自动清理策略（需手动或自配定时任务）
