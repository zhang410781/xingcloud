# 知识图谱与业务上下文 — 产品工作流

## 1. 产品目标

知识图谱是 SRE 数字员工的"空间认知能力"。它把 CMDB 资产、可观测性数据源、告警规则之间的关系编织成一张拓扑网，让故障研判智能体和巡检智能体知道"哪个主机上跑了哪个服务""这个告警影响了谁的什么"。

关键原则：知识图谱不创建、不猜测、不复制 CMDB 资产或数据源，只引用明确登记的数据。

## 2. 核心概念

| 概念 | 说明 |
|------|------|
| **一级资产分组** | 用户自定义的业务归属标签（如：支付业务、智能平台） |
| **业务上下文** | 知识图谱的工作单元：选择一个资产分组 + 一个指标源 + 一个日志源 + 一个环境编码 |
| **资产关联** | CMDB 中登记的 CI 关系（主机→集群、服务→主机等） |

## 3. 工作流

### 3.1 创建业务上下文

```
前置条件：
  ✅ CMDB 中已有一级资产分组
  ✅ 至少一个指标数据源和一个日志数据源已配置并连通

操作步骤：
  1. 知识图谱 → 新增业务上下文
  2. 选择一级资产分组
     （系统自动加载该分组关联的所有 CMDB 资产：主机/K8S/中间件）
  3. 选择一个指标数据源
  4. 选择一个日志数据源
  5. 填写唯一环境编码（如：prod-payment，作为告警和查询的标准标识）
  6. 保存

保存后效果：
  - 知识图谱自动建立资产-数据源-环境编码的关联关系
  - 告警规则 AI 研判时自动使用该上下文的指标源和日志源
  - 巡检智能体按业务上下文范围执行巡检
```

### 3.2 日常维护

```
资产变更后：
  CMDB 中修改资产分组 → 知识图谱自动感知（当前需手动刷新业务上下文）

数据源变更后：
  可观测性中更换数据源 → 编辑业务上下文重新选择

业务重组：
  创建新的一级资产分组 → 创建新的业务上下文
  旧业务上下文可保留（历史告警仍需引用）或归档
```

## 4. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| CMDB（资产+分组） | 业务上下文的资产列表 |
| 可观测性（数据源） | 业务上下文的指标源+日志源 |
| 业务上下文 | 故障研判智能体（数据源范围约束） |
| 业务上下文 | 巡检智能体（巡检范围） |
| 业务上下文 | 告警规则（环境编码注入） |

## 5. 边界与限制

- 不自动发现资产关联关系（需 CMDB 中手动定义 CI 关系）
- 不支持自定义图谱查询（只用于智能体范围约束，不可浏览全量关系图）
- 资产分组变更后不会自动触发业务上下文刷新
