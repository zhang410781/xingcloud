# CMDB 与资产登记

> 状态：当前已实现

## 1. 产品定义

CMDB 是业务上下文、巡检、告警研判和任务执行的资产范围底座。资产必须先登记，知识图谱只引用已存在的资产，不代替资产录入。

## 2. 当前入口

- `资产管理 / 资产登记`
- `资产管理 / 中间件资产`
- `平台管理 / K8S 集群`
- `可观测性 / 数据源`

## 3. 一级业务分组

管理员可以创建自定义一级业务分组，例如“智能平台”“支付业务”“中间件集群”。产品不强制拆分生产、测试、开发层级；当前业务上下文保存时统一按生产类型处理。

一个资产可以根据实际归属关联业务分组，中间件资产允许被多个业务分组使用。

## 4. 可登记资产

| 类型 | 主要信息 |
| --- | --- |
| 服务器 | 名称、地址、端口、认证信息、系统与连接状态 |
| K8S | 已在平台管理登记的集群引用 |
| 中间件 | 类型、名称、地址、端口、认证、说明和业务分组 |

中间件没有真实资产时显示空态，不创建演示或伪造数据。支持的登记类型以当前表单选项为准，监控看板中 MySQL、PostgreSQL、Redis 和 Kafka 依赖对应 Exporter 指标。

## 5. 状态说明

- **未检测**：尚未执行连接检测或刷新，不代表连接失败。
- **连接成功**：最近一次检测通过。
- **连接失败**：最近一次检测返回错误，应检查网络、端口和认证。
- 资产名称旁的状态标记应结合最近检测信息理解，不能只按颜色判断。

## 6. 标准工作流

1. 创建一级业务分组。
2. 登记服务器和中间件资产。
3. 在平台管理中接入 K8S 集群。
4. 将 K8S 集群登记为该业务分组下的 K8S 资产。
5. 在可观测性中登记 Prometheus 和日志源。
6. 到知识图谱配置中创建业务上下文并引用这些资源。

## 7. 当前边界

- CMDB 不自动发现所有基础设施。
- 平台不会因创建业务上下文而自动创建资产。
- 资产凭据仅用于平台连接和受控任务；文档和通知不得输出真实密码。
