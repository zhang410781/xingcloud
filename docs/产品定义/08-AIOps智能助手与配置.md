# AIOps 智能助手与配置

> 状态：当前已实现

## 1. 智能助手

智能助手支持独立页面和全局浮层。每个会话可以绑定业务上下文，后续查询使用该上下文对应的资产、Prometheus、日志源和 K8S 集群。

对于“集群有几个节点”“巡检当前集群”“排查异常 Pod”等问题，平台优先走确定性处理流程，不依赖模型主动生成 tool call。

## 2. 执行状态与中断

消息处理状态包括 pending、running、completed、failed 和 cancelled。生成期间可点击“停止生成”；取消后的晚到结果不能覆盖已取消状态。

单次处理总时限为五分钟。超时后保留已经采集的部分证据和未完成原因，允许用户继续提问。

## 3. Action

Action Registry 定义问题入口、所需上下文、风险级别、处理模式、默认 Skill 和输出块。它是任务路由与安全控制，不是独立 Agent 进程。

## 4. Skill

Skill 是专业方法和工具依赖的能力包。平台提供内置 Skill，也允许创建、编辑、克隆团队 Skill。实际可用工具仍受 MCP 可用性、用户 RBAC 和 Action 安全策略过滤。

## 5. MCP

- 平台 MCP 清单将现有平台查询能力暴露为受控工具。
- 外部 MCP 可以配置地址、鉴权、工具清单和运行边界。
- 工具调用记录进入智能体审计。

MCP 不绕过平台 API、权限和确认流程。

## 6. 模型供应商

管理员可配置兼容模型供应商的 Base URL、API Key、默认模型、备用模型、超时和上下文消息数，并执行连接测试和模型发现。

模型不可用时，确定性巡检和告警证据仍可返回部分结果；需要自然语言综合的普通问答会显示模型配置或调用错误。

## 7. 输出与安全

- 平台支持 Markdown 和原生报告块。
- 写操作先生成待确认动作，再进入任务或工单流程。
- 不展示模型思维链。
- 敏感字段在证据和通知中脱敏。

## 8. 当前边界

- 当前不是七个独立智能体组成的微服务系统。
- 模型不能创造平台未采集的指标、日志、Trace 或拓扑事实。
- 自愈仅能作为建议或待确认草稿，不能无人值守执行。
