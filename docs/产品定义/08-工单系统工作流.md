# 工单系统 — 产品工作流

## 1. 产品目标

工单系统将 SRE 的变更操作纳入审批流程。告警分析后的修复建议如需执行变更（发布、重启、调整配置），应通过工单系统流转并获得审批，确保操作可追溯、可回滚。

## 2. 工单类型

| 类型 | 说明 | 审批流 |
|------|------|--------|
| 应用发布 | 服务版本变更、配置更新 | 需要审批：技术负责人 → 架构师 |
| 事务工单 | 批量变更、故障恢复操作 | 需要审批：值班主管 |

## 3. 工作流

### 3.1 应用发布工单

```
前置条件：
  ✅ 目标资源已在 CMDB 中登记
  ✅ 部署任务已准备好（在任务中心预定义）

操作步骤：
  1. 提交发布申请
     → 填写应用名、版本号、变更内容、目标资源
     → 选择审批流
     → 提交

  2. 审批流转
     → 通知审批人（企微/飞书/邮件）
     → 审批人查看变更内容
     → 批准 / 驳回 / 打回补充

  3. 执行发布
     → 审批通过后，申请人在任务中心选择预定义的部署任务
     → 确认执行
     → 系统记录执行结果

  4. 结果确认
     → 申请人确认发布成功/失败
     → 失败则回滚/重新执行
     → 工单关闭
```

### 3.2 事务工单

```
适用于：故障恢复中的变更操作、批量重启、紧急修复

流程：提交 → 审批（可加急） → 执行 → 确认
审批流：建议设置紧急通道（超时自动通过？当前不支持）
```

## 4. 审批流配置

管理员在"审批流"页面配置：

```
审批流名称 → 审批节点列表（角色/用户） → 保存
```

示例：
```
"标准发布审批"：
  节点1：技术负责人（审批）
  节点2：架构师（审批）
  节点3：发布执行人（确认执行结果）
```

## 5. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 任务中心（部署任务） | 发布工单的执行阶段 |
| 用户（变更内容） | 审批人通知 |
| 审批结果 | 任务中心执行触发 |

## 6. 边界与限制

- 当前不支持工单自动生成（告警→自动创建工单）
- 不支持多级并行审批
- 不支持加急审批超时自动通过
- 不提供工单 SLA 统计（已超时未审批）
