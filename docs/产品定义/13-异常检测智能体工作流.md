# 异常检测智能体 — 产品工作流

## 1. 产品目标

异常检测智能体是告警系统的"前置哨兵"。它不依赖人工配置阈值，而是通过多种统计算法（3-Sigma、EWMA、IQR、Isolation Forest）对指标历史数据做异常评分，在固定阈值告警触发前发现"指标在偏离基线但还没超限"的隐患。

它与告警智能体的关系：**异常检测发现趋势异常 → 提示运维关注 / 自动创建低级别告警 → 故障研判智能体可选分析**。

## 2. 算法说明

| 算法 | 适用场景 | 输入 | 输出 |
|------|---------|------|------|
| **3-Sigma** | 正态分布的指标（如 CPU、内存） | 历史值 + 当前值 | 是否超出均值±3σ |
| **EWMA** | 有趋势/噪声的指标（如请求量、错误率） | 历史值 + 当前值 + 平滑系数 α | 加权移动平均偏离度 |
| **IQR** | 非正态分布的指标（如延迟 P99） | 历史值 + 当前值 | 是否超出 Q1-1.5IQR 或 Q3+1.5IQR |
| **Isolation Forest** | 多维/复杂模式的指标 | 历史值 + 当前值 + 随机树数量 | 异常概率评分 |

**最终判决**：多算法投票机制（`detect_anomaly(values, min_votes=2)`），至少 2 个算法同时判定异常时才输出异常。

## 3. 工作流

### 3.1 定时异常扫描（被动模式）

```
按配置周期执行（默认每 15min）
   → 从 Prometheus 拉取目标指标的最近 24h 历史数据
   → 对每条时间序列执行多算法检测
   → 汇总结果：
      - 正常：不处理
      - 异常（2/4 算法投票通过）：
          → 生成异常记录（含评分、异常方向、算法投票明细）
          → 可选：创建 info/warning 级别告警事件
          → 可选：触发故障研判智能体做简要分析
```

### 3.2 集成到告警规则（主动模式）

```
用户在创建告警规则时，可选择"智能检测"替代固定阈值：
  → 规则定义中勾选"启用智能异常检测"
  → 无需配置阈值数值，只需选择数据源 + 查询语句 + 持续时间
  → 评估时：
      → 后端自动获取历史数据
      → 执行多算法投票
      → 投票通过 → 触发告警
      → 投票未通过 → 跳过
```

## 4. 异常事件结构

```json
{
  "metric": "node_cpu_usage_percent",
  "labels": {"instance": "10.0.1.1", "job": "node_exporter"},
  "current_value": 92.5,
  "baseline": {
    "mean": 45.2,
    "stddev": 8.1
  },
  "algorithms": [
    {"name": "3-Sigma", "is_anomaly": true, "score": 5.84, "threshold": 3.0},
    {"name": "EWMA", "is_anomaly": true, "score": 4.2, "threshold": 3.0},
    {"name": "IQR", "is_anomaly": false, "score": 2.1, "threshold": 1.5},
    {"name": "IsolationForest", "is_anomaly": true, "score": 0.82, "threshold": 0.65}
  ],
  "votes": 3,
  "min_votes_required": 2,
  "direction": "up",
  "anomaly": true
}
```

## 5. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 可观测性-指标数据源 | 异常检测算法的数据输入 |
| 异常检测结果 | 告警引擎（生成异常事件） |
| 告警引擎 | 故障研判智能体（可选分析） |
| 告警规则（智能模式配置） | 触发异常检测的评估上下文 |

## 6. 常见场景

| 场景 | 说明 |
|------|------|
| 某服务请求量在 30min 内持续上涨但未超固定阈值 | 异常检测发现趋势偏离 → 创建 info 告警 → 提醒关注 |
| 配置了固定阈值的告警规则频繁误报（如业务高峰自然波动） | 改用智能检测模式，由算法自动适应基线 |
| 新接入的主机/中间件未配置任何告警规则 | 开启默认异常扫描兜底，防止完全无监控覆盖 |

## 7. 边界与限制

- 当前仅支持 Prometheus 指标数据源的异常检测，不支持日志
- 算法投票数 >= 2 才判定异常，可能存在漏报（单算法强信号但未达投票门槛）
- 异常检测不自动创建工单或升级通知，仅生成告警事件依赖告警智能体分发
- 历史数据窗口固定为 24h，不支持自定义基线周期
