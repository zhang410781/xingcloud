# CMDB 资产登记 — 产品工作流

## 1. 产品目标

CMDB 是 SRE 数字员工的"记忆库"。所有智能体（巡检、故障研判、日志分析、告警）依赖 CMDB 知道"要查什么""什么和什么有关"。没有 CMDB，智能体无法做范围约束——不知道集群包含哪些主机、服务部署在哪、异常影响谁。

## 2. 前置条件

- 用户拥有 `ops.task.resource.manage` 或 `ops.host.manage` 权限
- K8S 集群可连通（通过 kubeconfig 引导工具创建只读 ServiceAccount）
- 需要登记的主机可 SSH 连通（非强制，但影响任务执行）

## 3. 角色与权限

| 角色 | 可操作范围 | 权限码 |
|------|-----------|--------|
| 平台管理员 | 全部 CMDB 管理 | `ops.task.resource.manage` |
| 运维 | 查看资产、登记主机 | `ops.task.resource.view` + `ops.host.manage` |
| 研发 | 只读查看 | `ops.task.resource.view` |

## 4. 工作流

### 4.1 冷启动：首次接入平台

```
创建一级业务分组                    ← 如"支付业务""智能平台""中间件集群"
   │
   ▼
登记基础资产                        ← 主机 / K8S 集群 / 容器环境
   │
   ▼
登记中间件资产                      ← MySQL / Redis / Kafka / PostgreSQL
   │
   ▼
配置数据源（见可观测性工作流）       ← 指标源 + 日志源
   │
   ▼
创建业务上下文                      ← 选择资产分组 + 指标源 + 日志源 → 知识图谱可用
```

### 4.2 日常维护

```
新增主机/集群
   → 填写 IP/SSH/kubeconfig 信息
   → 系统自动探测可用中间件（可选）
   → 关联到已有业务分组
   → 知识图谱自动同步绑定关系
```

```
资产变更
   → 修改资产标签/分组/负责人
   → 知识图谱刷新关联
   → 影响面评估：关联的告警规则、巡检范围是否需要调整
```

### 4.3 资产关系管理

CMDB 支持多级资源树结构，例如：

```
支付业务（一级分组）
  ├── 生产环境
  │   ├── 主机 10.0.1.1
  │   ├── 主机 10.0.1.2
  │   ├── K8S 集群 production-cluster
  │   └── Redis 集群 payment-redis
  └── 中间件
      └── MySQL production-mysql-01
```

## 5. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 用户手动录入 / K8S API 导入 | 知识图谱（业务上下文） |
| SSH 探测中间件 | 任务中心（目标资源列表） |
| | 告警规则（监控对象选择） |
| | 巡检智能体（巡检范围） |

## 6. 常见场景

| 场景 | 操作路径 |
|------|---------|
| 接入一个新的 K8S 集群 | 平台管理 → 新增集群 → 贴 kubeconfig 引导命令 → 等待连通 |
| 发现一台服务器需要纳入监控 | 资产登记 → 新增主机 → 填入 IP/SSH → 关联业务分组 |
| 中间件升级后需要更新资产 | 中间件资产 → 编辑连接信息 → 测试连通 |
| 业务重组需要调整分组 | 资产登记 → 编辑分组 → 批量迁入/迁出资产 |

## 7. 边界与限制

- 当前不自动发现资产（不扫描网段、不监听注册中心）
- 不支持多云资源自动同步（AWS/GCP/Azure）
- 资产历史变更记录需通过审计追溯，CMDB 本身不维护版本历史
