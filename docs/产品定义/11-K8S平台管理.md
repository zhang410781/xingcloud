# K8S 平台管理

> 状态：当前已实现

## 1. 产品定义

K8S 平台管理负责接入已有 Kubernetes 集群并提供资源查看、日志、终端和受控操作。多个集群分别登记；业务上下文通过 CMDB 中的 K8S 资产决定使用哪个集群。

## 2. 集群接入

管理员在 `平台管理 / K8S 集群` 中填写集群名称、API Server、kubeconfig 等连接信息并执行验证。平台不会假定当前环境只有一个全局集群。

## 3. 资源能力

当前代码覆盖集群概览、Node、Namespace、Pod、Deployment、StatefulSet、DaemonSet、Service、Ingress、ConfigMap、Secret、PVC、事件、日志和部分配置资源版本/回滚操作。

具体按钮由用户权限、资源类型和集群连接状态决定。

## 4. 与其他模块关系

- CMDB：将集群登记为一级业务分组下的 K8S 资产。
- 业务上下文：从资产分组自动解析集群。
- 可观测性：Prometheus 提供指标，K8S API 提供对象状态。
- 告警研判：读取目标 Pod、Warning Event、当前与 previous 日志。
- 巡检：生成集群、节点、工作负载和存储证据。
- 任务中心：承接受控 K8S 操作。

## 5. 当前边界

- 平台不通过名称相似度静默选择其他集群。
- 集群连接失败时只影响对应上下文和查询。
- Docker 容器环境不是当前平台管理菜单的一部分。
