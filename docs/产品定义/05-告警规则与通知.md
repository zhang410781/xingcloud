# 告警规则与通知

> 状态：当前已实现

## 1. 告警规则

平台将内置定义作为规则模板，用户选择模板和指标数据源后生成可执行规则实例。实例可以单独修改阈值、持续时间、执行周期、标签、查询和启用状态。

Prometheus 规则必须明确绑定指标数据源。正式执行不因配置缺失而回退到默认 Prometheus。

## 2. 创建方式

- 从内置模板实例化。
- 使用表单化指标、条件和阈值创建。
- 高级模式填写 PromQL 和触发条件。

保存前应先试运行。试运行返回匹配序列、触发对象和错误；无匹配不等于规则异常，可能是当前窗口内没有符合查询的时序数据。

## 3. 规则分类

内置K8S规则目录以原五个规则包、52条来源告警为覆盖基线，合并双阈值后形成46条规范化模板，按 API Server、工作负载、节点网络、K8S存储和节点系统展示。服务器、数据库和中间件规则包继续独立展示。模板存在不代表目标环境已经安装相应 exporter。

Agent1-4 目录是当前唯一的内置 K8S 模板来源。同步目录时会删除已被替代的旧 K8S 模板及其生成实例，不再以“停用但保留”的方式展示；无模板归属的用户自定义规则不在清理范围内。

每条K8S模板保存来源规则名、PromQL、标签维度、单位、Warning/Critical阈值、持续时间、摘要、详细描述和证据采集Profile。Info事件默认不通知、不研判。新实例默认禁用，需试运行确认后启用。

## 4. 运行质量

规则详情记录最近执行时间、执行耗时、查询错误、无数据次数、触发次数、抖动次数和最近命中对象。

“无数据 75 次”表示规则连续或累计执行中有 75 次没有返回可评估样本，不等于触发了 75 次告警。应检查指标名、标签、数据源、时间范围和 exporter。

## 5. 告警隔离

规则、告警指纹、聚合键和通知记录包含业务上下文及指标数据源维度，避免不同集群的同名规则串告警、串去重或串通知。

## 6. 通知资源

### 全局资源

- 通知渠道。
- 接收人。
- 接收组。
- 巡检报告计划。

接收人由名称和各渠道联系方式组成，界面不要求填写平台用户 ID。

### 通知策略

策略按业务上下文、数据源、集群、命名空间、服务、级别和自定义标签匹配，并支持：

- 优先级与继续匹配。
- 分组等待、分组间隔和重复通知。
- 静默时间表。
- 抑制规则。
- 升级步骤。
- 首次、恢复和研判完成通知。

飞书机器人渠道要求配置签名密钥。策略预览用于确认实际命中的渠道、接收资源、聚合和静默结果。

## 7. 通知阶段

1. 首次触发：发送简洁告警摘要。
2. 精准研判完成：仅当本活动周期的首次告警已实际送达时，按同一通知策略和已送达渠道发送结构化研判更新。
3. 告警恢复：发送恢复通知。

普通通知不堆叠完整 PromQL、原始日志或模型思维过程。

研判通知继承告警的静默、屏蔽、抑制、分组和风暴降噪结果。首次告警未实际发送、处于静默/抑制状态或在聚合中仅记录时间线时，研判仍正常执行并保存到告警详情，但不会绕过降噪策略单独发送。重新触发后的新活动周期只继承该周期的首次通知记录，不复用历史周期的发送结果。

首次通知使用安全模板渲染 `{{ $labels.xxx }}`、`{{ $value }}`、`printf` 和 `humanizePercentage`。缺失标签显示 `-`，模板诊断不阻塞规则执行。通知提供“查看详情、认领、屏蔽”入口；认领和屏蔽必须先进入登录后的确认页，屏蔽默认仅针对当前指纹一小时。

内置“Agent-4 标准通知策略”按业务上下文、数据源、命名空间和规则编码聚合，采用10秒分组等待、1分钟组间隔和12小时重复间隔；同对象 Critical 抑制 Warning/Info，Warning 抑制 Info。预设不会覆盖已有策略，应用时仍需选择通知渠道和接收组。
