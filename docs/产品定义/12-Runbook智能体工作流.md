# Runbook 智能体 — 产品工作流

## 1. 产品目标

Runbook 智能体将 SRE 的故障处理经验沉淀为可复用、可搜索、可执行的运维剧本。

故障研判智能体定位出根因后，Runbook 智能体自动匹配最相关的处理步骤推荐给值班 SRE；SRE 也可以主动搜索 Runbook 库，查看当前告警的最佳处理方式。已发布的 Runbook 支持一键在任务中心执行。

## 2. Runbook 数据模型

| 字段 | 说明 |
|------|------|
| 标题 | 可读标识，如"MySQL 连接数过高处理流程" |
| 标签 | 用于智能匹配的标签集（关联告警标签、资产类型、级别） |
| 内容 | 结构化步骤 Markdown（可包含脚本引用、命令模板） |
| 关联任务 | 可引用任务中心预定义的执行脚本 |
| 关联 Session | 可引用 AIOps 对话作为创作来源 |
| 版本 | 每次编辑生成新版本，可回溯历史 |
| 状态 | draft / published / archived |
| 创建/更新人 | 记录运维知识贡献者 |

## 3. 工作流

### 3.1 创建 Runbook（写知识）

```
用户进入 Runbook 管理
  → 新建 Runbook
  → 填写标题 + 标签 + 结构化内容（步骤式 Markdown）
  → 可选：关联任务中心的预定义脚本（供一键执行）
  → 保存为 draft

来自 AIOps 会话：
  AIOps 对话 → "保存为 Runbook" → 自动填充故障上下文作为内容草稿
  → 用户补充完整操作步骤 → 发布
```

### 3.2 发布与版本管理

```
draft → 编辑完成 → 发布（published）
published → 发现错误 → 编辑 → 新版本自动生成
published → 不再适用 → 归档（archived）

每次编辑：
  - 版本号递增
  - 保留历史版本（versions API）
  - 可回溯任何历史版本
```

### 3.3 智能匹配（故障处置场景）

```
故障研判智能体输出根因
  → Runbook 智能体以告警标签 + 根因摘要为输入
  → 在 Runbook 库中检索匹配（标签匹配 + 内容关键词）
  → 返回 Top-3 推荐列表（带置信度排序）
  → 随通知消息一并发送给 SRE
  → SRE 在消息/Web 工作台上直接查看
```

### 3.4 执行 Runbook（阅读 → 行动）

```
SRE 打开 Runbook 详情
  → 阅读步骤（图文混排的 Markdown）
  → 如有关联的任务脚本：点击"一键执行"→ 跳转到任务中心预览
  → 手动确认后执行
  → 执行结果自动关联回该 Runbook 的最近一次使用记录
```

## 4. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| 故障研判（根因结果） | Runbook 匹配的触发输入 |
| 用户手动创建/编辑 | Runbook 库内容 |
| Runbook 引擎 | 任务中心（关联的脚本执行） |
| Runbook 匹配结果 | 通知消息（随告警发送给 SRE） |

## 5. 常见场景

| 场景 | 说明 |
|------|------|
| MySQL 连接数过高告警 → 自动匹配"MySQL 连接数处理流程" | SRE 一键查看步骤，定位慢查询源头 |
| 新运维入职 → 在 Runbook 库中检索"K8S Pod CrashLoopBackOff 处理" | 学习团队积累的故障处置经验 |
| 故障复盘 → 追溯该故障使用了哪个 Runbook，执行结果如何 | 沉淀团队 SOP，持续优化 |

## 6. 边界与限制

- Runbook 内容当前为手动编写，不支持从 AIOps 对话自动生成完整内容（需人工确认后发布）
- 不提供 Runbook 执行进度追踪（当前只记录"被查看/被执行"）
- 不提供 Runbook 执行结果自动回写（需 SRE 手动更新 Runbook 内容）
