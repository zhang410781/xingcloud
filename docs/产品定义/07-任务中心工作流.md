# 任务中心 — 产品工作流

## 1. 产品目标

任务中心是 SRE 数字员工的"双手"。它负责在目标主机或 K8S Pod 上执行预定义脚本、批量命令或 Ansible Playbook。巡检智能体依赖任务中心执行巡检脚本，故障分析后如需执行诊断命令也通过任务中心完成。

## 2. 任务类型

| 类型 | 执行目标 | 输入 | 输出 |
|------|----------|------|------|
| 批量命令 | CMDB 中的主机 | Shell 命令字符串 | 命令回显（stdout/stderr） |
| Ansible Playbook | CMDB 中的主机 | Playbook 文件 | Ansible 执行结果 |
| 定时任务 | 已配置的主机 | 脚本 + 调度规则（Cron） | 每次执行的记录 |
| K8S 任务 | K8S Pod | kubectl 命令或 Job YAML | 执行结果 |

## 3. 工作流

### 3.1 即时任务执行

```
用户进入 任务工作台
   → 选择执行模式：单机 / 批量
   → 选择目标主机或主机组（来源：CMDB 资产列表）
   → 选择执行方式：Shell / Ansible
   → 输入脚本/命令
   → 预览确认（显示将要执行的目标+命令）
   → 确认执行
   → 后端 SSH 连接目标 → 执行脚本 → 实时流式回显
   → 任务完成后，用户可查看完整执行记录
```

### 3.2 计划任务配置（巡检调度）

```
用户进入 计划任务
   → 新增计划任务
   → 指定名称 + 调度规则（Cron 表达式 / 预置：每天/每周/每月）
   → 选择目标范围 + 脚本
   → 保存

执行时：
   按 Cron 触发 → 系统自动执行 → 记录执行结果
   执行失败 → 可配置告警通知
```

## 4. 任务资源管理

任务可执行的目标资源在 CMDB 中登记后，可在任务中心直接选取。支持：

- 主机分组（按 CMDB 一级业务分组过滤）
- 主机标签筛选
- 手动输入目标 IP（临时执行）

## 5. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| CMDB（主机列表） | 任务目标选择范围 |
| 用户输入的脚本/命令 | 执行结果 → 审计记录 |
| 计划任务调度 | 巡检智能体（触发巡检） |

## 6. 边界与限制

- 当前不支持分布式批量执行（SaltStack 模式）
- 不支持结果 Webhook 回调（执行完毕→触发下游动作）
- 执行用户固定为 CMDB 中配置的 SSH 用户，不支持执行时切换用户
- 不支持 playbook 参数化模板
