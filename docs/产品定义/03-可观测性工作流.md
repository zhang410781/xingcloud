# 可观测性 — 产品工作流

## 1. 产品目标

可观测性是 SRE 数字员工的"眼睛和耳朵"。它为 7 个智能体提供实时/历史数据：巡检智能体靠它获取健康指标，故障研判智能体靠它获取指标证据和日志证据，日志分析智能体靠它搜索日志，异常检测智能体靠它获取指标历史，变更智能体靠它做指标对比。没有可观测性数据，智能体什么也看不见。

## 2. 数据源定义

平台支持两类数据源，统一在"平台总览"中管理：

### 2.1 指标数据源

| 属性 | 说明 |
|------|------|
| 类型 | Prometheus 兼容接口（PromQL） |
| 配置要素 | URL、认证方式（Basic/Token/无）、查询超时 |
| 当前覆盖 | K8S 集群内 Prometheus、独立 Prometheus 实例 |
| 使用范围 | 监控看板、指标查询、告警规则（PromQL 条件）、AIOps 指标证据包 |

### 2.2 日志数据源

| 属性 | 说明 |
|------|------|
| 类型 | Loki / ELK（Elasticsearch）/ ClickHouse |
| 配置要素 | URL、认证方式、索引/集合名（ClickHouse 可配多集合） |
| 日志类型 | 容器日志、K8S 集群事件、Ingress 访问日志、服务日志 |
| 使用范围 | 日志查询、告警规则（关键字/阈值条件）、AIOps 日志证据 |

### 2.3 数据源配置工作流

```
用户进入 平台总览
   │
   ├── 添加指标数据源
   │   → 填写 Prometheus URL + 认证信息
   │   → 点击"测试连接"
   │   → 验证通过，保存
   │
   └── 添加日志数据源
       → 选择类型（Loki/ELK/ClickHouse）
       → 填写 URL + 认证 + 集合配置（ClickHouse 可配置多个日志集合）
       → 测试连接
       → 保存
```

## 3. 监控看板工作流

当前平台提供 3 类原生监控看板（不依赖 Grafana）：

| 看板 | 数据来源 | 展示内容 |
|------|----------|---------|
| 服务器看板 | 默认指标源 | CPU/内存/磁盘/网络 |
| K8S 集群看板 | 默认指标源 | Pod 状态/Node 资源/集群事件 |
| 日志看板 | 默认日志源 | 日志量趋势/错误率/关键字分布 |

看板使用"平台总览中设定的默认数据源"。用户如需查看不同数据源，可通过指标/日志查询页面手动切换。

## 4. 指标查询工作流

```
用户进入 指标查询
   → 选择数据源（下拉选择已登记的 Prometheus）
   → 输入 PromQL 查询语句
   → 选择时间范围（预设：最近1h/6h/24h/自定义）
   → 选择步长
   → 执行查询
   → 返回图表 + 数据表格
```

**权限**：需要 `ops.metric.query` 权限码。

## 5. 日志查询工作流

```
用户进入 日志查询
   → 选择日志数据源（Loki/ELK/ClickHouse）
   → 输入查询语句（Lucene/KQL/SQL 取决于数据源类型）
   → 选择时间范围
   → 可选：添加过滤标签（namespace/pod/level）
   → 执行查询
   → 返回日志列表 + 时间分布柱状图
```

**权限**：需要 `ops.log.query` 权限码。

## 6. 模块间依赖

| 输入来源 | 输出去向 |
|----------|----------|
| Prometheus / Loki / ELK / ClickHouse | 监控看板 / 指标查询 / 日志查询 |
| 指标数据源 | 告警规则（PromQL 条件定义） |
| 日志数据源 | 告警规则（关键字/阈值条件） |
| 指标+日志数据源 | AIOps 证据包（故障研判智能体） |
| 数据源+业务上下文 | 知识图谱（范围绑定） |

## 7. 边界与限制

- 当前不接入 APM/Trace 系统（Jaeger/SkyWalking/Tempo）
- 不提供自定义看板编辑器（拖拽式）
- 不支持看板导入（Grafana JSON 或 Nightingale JSON 需手动处理）
- 日志数据源类型当前固定为 Loki/ELK/ClickHouse，不提供自定义 SQL 数据源
